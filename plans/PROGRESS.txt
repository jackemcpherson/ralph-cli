## 2026-01-22 - US-003

**Story:** Create FixLoopService for fix attempt logic

### What was implemented
- Created `src/ralph/services/fix_loop.py` with FixLoopService Pydantic model class
- `attempt_fix()` method invokes ClaudeService to fix a single finding using a structured prompt
- `run_fix_loop()` iterates through findings with retry logic (configurable max_retries, default 3)
- `_commit_fix()` creates individual commits using format: `fix(review): [reviewer] - [finding-id] - [description]`
- `_log_fix_success()` and `_log_fix_failure()` append structured entries to PROGRESS.txt
- Failed fixes after exhausting retries are logged and skipped, allowing subsequent findings to be processed
- `FixResult` NamedTuple tracks finding_id, success, attempts, and error

- `test_logs_success_to_progress` - Verifies successful fixes are logged to PROGRESS.txt
